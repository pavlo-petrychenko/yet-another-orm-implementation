# Вступ

**Орієнтовний обсяг:** 2-3 сторінки (ліміт методички — не більше 3).

## Що має містити

За методичкою, вступ послідовно описує:

1. **Актуальність роботи** — зростання TypeScript-екосистеми у бекенд-розробці, недоліки існуючих ORM щодо end-to-end типобезпеки, потреба у мультидіалектних рішеннях.
2. **Підстави для виконання** — практичний інтерес до проєктування шарованих архітектур, академічна спрямованість.
3. **Мета роботи** — спроєктувати та реалізувати типобезпечну ORM-бібліотеку для TypeScript із підтримкою декількох SQL-діалектів та порівняти її з існуючими рішеннями.
4. **Завдання** (6-7 пунктів) — проаналізувати предметну область; обрати архітектурні патерни; спроєктувати модульну структуру; реалізувати DML/DDL пайплайни; реалізувати подвійний публічний API; реалізувати міграційну підсистему; виконати порівняння з TypeORM/Objection.js/Drizzle; розробити демонстраційний застосунок.
5. **Об'єкт дослідження** — інструменти об'єктно-реляційного відображення у TypeScript.
6. **Предмет дослідження** — методи забезпечення end-to-end типобезпеки та мультидіалектності в ORM-бібліотеках.
7. **Методи й засоби** — порівняльний аналіз, проєктування шарової архітектури, прототипування, інтеграційне тестування (Jest + testcontainers).
8. **Сфери застосування** — серверні застосунки на Node.js, що потребують типобезпечного доступу до реляційних БД.
9. **Взаємозв'язок з іншими роботами** — позиціонування YAOI відносно TypeORM, Objection.js, Drizzle.
10. **Апробація та публікації** — публікація проєкту з відкритим кодом, посилання на репозиторій.
